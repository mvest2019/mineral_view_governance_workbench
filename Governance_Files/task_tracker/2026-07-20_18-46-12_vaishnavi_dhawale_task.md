## Task Description

Conducted a complete usability, design, and functional review of both websites (mineralview-marketing.vercel.app and mineralview-owner-demo.vercel.app) on desktop and mobile. All pages, buttons, forms, searches, filters, and navigation flows were tested using automated browser scripts, with screenshots and console/network logs captured for evidence.

Deliverables:

1. Full review report covering first impressions, UX, UI, mobile, content, accessibility, performance, trust, and conversion.
2. A prioritized list of 29 issues, each documented with: Current Problem, Desired Change, Requirements, and Acceptance Criteria.

Key Findings (Priority Order):

1. Duplicate codebases — The two sites run different versions of the same product (different menus, account menu, page versions, and demo data). A single shared codebase should serve both.
2. Record search broken — The "Find your record" search returns no response when used, and freezes the browser when submitted empty. This is the primary customer flow.
3. Operator comparison tool broken — All dropdown options show the same operator (Pioneer), so the tool compares a company against itself.
4. Dead button — "Start Pro — $299.95/mo" on the map page produces no action.
5. Inconsistent statistics — County counts (254 / 280 / 281), owner totals, well counts, and portfolio values differ across pages. All figures must come from a single generated data source.
6. Contact form failure — Submits successfully with all fields empty, displays "(prototype)" in the confirmation, and has no confirmed delivery.

Additional Issues Logged: Broken video thumbnails, internal developer notes visible to customers, stuck tooltip overlay, wrong lease shown on Production & Forecast, checkout copy contradictions, accessibility gaps (unlabeled fields, non-keyboard-accessible controls), and mobile layout defects (overlapping elements, undersized tap targets).
