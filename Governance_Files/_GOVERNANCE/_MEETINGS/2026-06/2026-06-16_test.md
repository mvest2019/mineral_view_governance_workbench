## Summary

Mineral View is a Texas oil & gas **mineral-rights intelligence platform**. It gives
mineral owners (and operators/investors) insight into their leases, wells, production,
and the activity happening around them, sold through subscription tiers.

The platform is six active repositories across three layers:

| Layer | Repos |
|---|---|
| Customer-facing web | Mview-Presentation-Next |
| Public / product APIs | PresentationSiteAPI (2 branches), MviewMapAPI, MViewPortalAPI |
| Internal tools | Mview-Cerebro-web |

Shared data stores: **PostgreSQL + PostGIS** (spatial lease/well data), **MongoDB**
(documents/content), **Redis** (caching). Payments via **Braintree**. Maps via **ArcGIS**.
Charts via **Highcharts / Chart.js**. Reports rendered as **PDF (Puppeteer/html-pdf)**
and **DOCX (docxtemplater)**. Deployed on **Windows Server / IIS** (`web.config`), Node
processes managed with **PM2**.

## 1. MViewPortalAPI

- **Observed purpose:** Main portal backend API for the logged-in owner experience.
- **Category:** Web / Portal API
- **Stack:** Node.js, Express, PostgreSQL + PostGIS (many `pg_functions` stored procedures), MongoDB, JWT auth, Braintree, Google APIs, nodemailer, docxtemplater.
- **Key route groups:** `auth`, `MasterData`, `MVestUser`, `mineralowner`, `LeaseSearch`, `gis` / `gishelper`, `WellAttachments`, `PotentialAcreage`, `Neighbor`, `MyPortfolio` / `UserPortfolio`, `FieldReport` / `FieldReportOwner`, `Community`, `Followers`, `Discussion`, `NewsFramework`, `notifications`, `SponsoredAdvertise`, `dashboard`, `ReportAbuse`, `Activity`.
- **Domain it owns:** mineral owners, leases, wells, portfolios, neighbor/offset activity, field reports, the in-app community.
- **Status:** ACTIVE — canonical portal backend.
- **Notes:** PostGIS stored functions handle spatial lease/well geometry and cashflow/disposition-volume updates (`update_dispvol_cashflow`).

## 2. MviewMapAPI

- **Observed purpose:** Dedicated API powering the interactive map (lease/well geospatial layers and on-map analytics).
- **Category:** GIS / Map API
- **Stack:** TypeScript, Express, PostgreSQL, MongoDB, PM2. Clean `controllers / services / models / routes` layout.
- **Key endpoints:** `getLeaseWellLocations`, `getproductionActivity`, `getnearbydevelopment`, `getOperatorActivity`, `getWellsByViewport`, `getleaseList`, `getwellslistbylease`, `getwellproductionChartData`, `searchByApiNo`, county/operator/field dropdowns, `play_reservior`, `claimedLeaseslist`.
- **Plan gating:** `featureAccess` middleware enforces subscription-tier limits (allowed vs. used counts, unlimited flags, plan id/name) per feature.
- **Status:** ACTIVE — the newest, best-structured API.
- **Notes:** Ships `COMPREHENSIVE-API-DOCS.md` and a `comprehensive-lease-data.json` fixture.

## 3. PresentationSiteAPI  *(two branches — see finding)*

- **Observed purpose:** Backend for the public presentation/marketing site: registration, subscriptions, payments, data purchase, and report generation.
- **Category:** Public Site API / Payments & Subscriptions
- **Stack:** Node.js, Express, PostgreSQL, MongoDB, Redis (operator caching), Braintree, Puppeteer / html-pdf, Turf/geolib (geo), node-cron.
- **Key route groups:** `auth`, `User` / `subscription`, `payment`, `enterprise-plan`, `email-verification`, `PurchaseData`, `ProductionData`, `MasterData`, `myportfolio`, `gis`, `pdfData`, `datacoverage`, `notification`, `internal`, `NewsFramework`, `utility`.
- **Background jobs:** cron-driven subscription management (incl. **downgrade** processing) and operator-cache warming/clearing.
- **Status:** ACTIVE — but exists as **two long-lived branches**: `Master` and `Mview_Prod`. Canonical branch must be confirmed (the `Mview_Prod` copy additionally carries a `db/` folder).

## 4. Mview-Presentation-Next

- **Observed purpose:** The public, customer-facing website — where owners view insights and manage their subscription.
- **Category:** Web / Storefront (customer-facing)
- **Stack:** Next.js 15 (App Router), React, TypeScript, ArcGIS (`@arcgis/core`), Highcharts, Braintree Drop-in, Tailwind + Radix UI, Cypress, NextAuth (`auth.ts`), OTP input.
- **Key route segments:** `(Map)/map` + `interactive-map-features`, `community` (groups, forums, feed, profile, bookmarks, notifications), `blogs` (field / operator / play-type), `manage-subscription`, `data-coverage`, `landman-mineral-management`, `build-social-network`, `contact-us`.
- **Status:** ACTIVE — primary customer surface.

## 5. Mview-Cerebro-web

- **Observed purpose:** Internal admin / analytics web app ("Cerebro").
- **Category:** Admin / Ops (internal)
- **Stack:** Angular 16 + Angular Universal (SSR via Express `server.ts`), Chart.js, Bootstrap, ngx-bootstrap.
- **Status:** ACTIVE — internal tool. Exact internal scope to confirm with Ryan.
