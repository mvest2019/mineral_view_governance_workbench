# Priority Question Answer

Question:
Can accessibility findings like keyboard access, focus order, and screen-reader behavior be signed off from static screenshots alone, or must the live app be re-tested?

Answer:
No. Accessibility findings such as keyboard navigation, focus order, focus visibility, screen-reader behavior, ARIA implementation, and interactive component accessibility cannot be fully validated from static screenshots alone. Screenshots can help identify visual issues (such as color contrast, missing labels, or layout concerns), but they cannot verify how the interface behaves during actual user interaction.

A live application must be re-tested to validate accessibility-related behavior. This should include keyboard-only navigation, screen-reader testing, focus management, interactive component behavior, semantic HTML/ARIA verification, and, where applicable, automated accessibility scans followed by manual testing. Therefore, any accessibility observations made from screenshots should be treated as preliminary and should not be considered a final accessibility sign-off until they have been verified in the live application.
