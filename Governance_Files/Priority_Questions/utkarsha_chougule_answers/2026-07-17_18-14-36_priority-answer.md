# Priority Question Answer

Question:
Is the $0-fallback being displayed as a real value a UI change or a data-source fix, and who owns it?

Answer:
Data-source fix (the real fix, P0): The root cause is upstream — valuations are stored as text and MVEstimate returns $0 for depleted/inactive leases, so the app substitutes the county-appraised number. That's a data/calculation defect, not a styling issue. Fixing it properly means correcting how valuations are stored and computed so MVEstimate produces a real number (or a true null) per lease. This is the substantive owner.
UI change (the interim/presentation part): Until the data is fixed, the UI shouldn't present a fallback as a real valuation. The display should show an honest non-numeric state ("No estimate — lease inactive") instead of borrowing the county figure, and drop the "$0-fallback shown" badge. That's a front-end change.
