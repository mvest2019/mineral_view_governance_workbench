# Priority Question Answer

Question:
Is there automated alerting for scan failures, or was the Gemini-timeout issue only caught by manual review?

Answer:
There's no automated alerting for scan failures. The scan engine has fallback logic — Gemini-primary with an OpenAI fallback, respecting free-tier limits — so it tries to recover on its own, but that's resilience, not alerting; nothing notifies us when a scan fails or degrades.

So the Gemini-timeout issue was caught by manual review, not by an alert — someone had to notice and dig in (the same way the OpenAI key-shadow problem was found by inspecting it, not flagged automatically).

Flag worth stating: reliable failure detection is a gap right now — it leans on the fallback chain plus us checking manually, so automated alerting on scan failures would be the improvement to make.
