# Priority Question Answer

Question:
What is "Trendelier", and what exactly do the scans check and get verified against?

Answer:
What Trendelier is: our own internal content tool. It scans sources per channel, generates insights and content drafts (reel/short-form scripts, long-form YouTube scripts, CTAs), and tracks social analytics for our brand channels. Basically it powers content research and planning across the different brands.

What the scans check: each scan pulls from that channel's configured sources — news/RSS feeds, YouTube trending, and structured data feeds (e.g. EIA and RRC/Texas data for the mineral side, direct precious-metals/jewelry publisher feeds for BOLD). It reads the actual article/video bodies, then an LLM scores each item for relevance to the channel before anything becomes an insight or draft.

What it's verified against:

Relevance — items must clear a pure LLM relevance gate (~70+) to qualify; thin or off-topic items get skipped rather than forced through.
Real content, not headlines — it fetches full bodies (some feeds only give redirect stubs, which are known and handled) so insights are built on actual source text, not just titles.
Freshness/validity — structured feeds skip preliminary/lagging data (e.g. RRC reporting-lag months) so it doesn't act on incomplete numbers.
Analytics side — published-post numbers are reconciled against the platforms' own API figures (YouTube Data API, Meta Graph API) and de-duplicated so counts reflect real activity.
