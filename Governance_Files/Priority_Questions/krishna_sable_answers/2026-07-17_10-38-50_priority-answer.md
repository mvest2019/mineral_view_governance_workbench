# Priority Question Answer

Question:
How stale can a dossier snapshot be before it must be regenerated, and who enforces that rule?

Answer:
A dossier is a dated snapshot: MVestimate/royalty/cashflow update daily, production monthly, prices constantly; ownership/geo/operator age slowly. Rule of thumb: regenerate if more than a few days old before it's actually used, and always regenerate after the 2025 CAD data lands. Enforcement today is manual, not automated — each file carries a freshness stamp in its header reminding whoever builds/sends it; the person generating the dossier is the enforcer. (If we want a hard gate, that's a process to add — it doesn't exist yet.)
