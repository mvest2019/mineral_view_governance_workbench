# Priority Question Answer

Question:
What now prevents duplicate or repeated sends when the scheduler retries stuck emails every 5 minutes?

Answer:
The dispatcher atomically claims rows with UPDATE … FOR UPDATE SKIP LOCKED (flips SCHEDULED→SENDING), so overlapping runs can't grab the same row. Capped at 3 attempts, with a 15-min stuck-row rescue — no double-sends.
